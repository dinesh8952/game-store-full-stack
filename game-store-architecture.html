Game Store

Architecture & Setup

Express.js TypeScript PostgreSQL + Prisma Redis Docker EJS JWT

Architecture

Docker Compose
Express App (Node.js)
Global Middleware
requestId — assigns crypto.randomUUID to every request
helmet — CSP, frameAncestors, security headers
cors — cross-origin (disabled in production)
bodyParser + cookieParser + methodOverride
setLocals — JWT → Redis user cache (60s TTL) → DB
↓
/auth
csrfMiddleware authLimiter controllers
/games
authenticate requireProfile requireApproved csrfMiddleware playLimiter (play) controllers
/admin
authenticate requireAdmin csrfMiddleware controllers
↓
Services → Prisma / Redis
Background Job — runs every 60s
Redis distributed lock (one replica only) → SCAN play_count:* keys → GETSET atomic capture → PostgreSQL update → restore on DB failure
PostgreSQL
  • User
  • UserProfile
  • Game
  • GamePlay
Redis
  • user_cache:{id} — 60s TTL
  • play_count:{id} — write buffer
  • dashboard:stats — 5min TTL
  • blacklist:{jti} — logout tokens
  • lock:play_count_flush
migrate service — runs prisma migrate deploy once before app starts. App depends on service_completed_successfully — prevents migration lock contention in multi-replica deployments.

Setup

1
Clone the repository
git clone <repo-url>
cd game-store
2
Set up environment variables
Copy .env.example → .env and fill in the values.
cp .env.example .env
3
Start everything
Migrations run automatically. Super admin is seeded on first boot.
docker compose up
Health check available at GET /health — returns DB and Redis status.
